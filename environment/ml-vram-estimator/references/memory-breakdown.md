# 显存分项总公式

> last updated: 2026-01。这是估算的"主表"。所有项相加 × 安全余量 = 每卡占用。

```
每卡显存 ≈ (权重 + KV缓存 + 梯度 + 优化器状态 + 激活) / 并行度 + 框架开销
          再 × 余量(默认 1.1,留碎片/分配器空间)
```

| 项 | 何时算 | 公式 | 字节/参或/元素 |
|----|--------|------|----------------|
| **权重** | 总是 | 参数量 × 每参字节 | bf16=2, fp8=1, int4/nvfp4=0.5(+scale 开销) |
| **KV cache** | 推理(训练也有但被激活盖过) | 见 inference-and-kv.md | 2×kv_heads×head_dim×层数×B×S×字节 |
| **梯度** | 训练 | 参数量 × 字节 | bf16=2(fp32 主权重另算在优化器) |
| **优化器状态** | 训练 | 参数量 × 每参字节 | **Adam=12**(fp32 主拷4+动量4+方差4), SGD=8, Adafactor≈4 |
| **激活** | 训练为主 | 见 training-memory.md | 随 B×S×hidden×层数,长 seq 有 S² 项 |
| **开销** | 总是 | CUDA context + 框架缓冲 | 经验 ~0.5–2GB/卡 |

## 每参字节速记(混合精度训练全量微调)

```
权重 bf16        2
梯度 bf16        2
优化器 Adam     12   ← 最大头!
─────────────────
合计           16 字节/参(还没算激活)
```

→ 经验法则:**全量微调 ≈ 16 × 参数量(GB,按十亿参)+ 激活**。
例:7B 全量微调 ≈ 16×7 = 112GB(权重+梯度+优化器)+ 激活,单卡 80G 必爆。
推理只需 **权重 + KV cache**,小得多。

## dtype 字节表

| dtype | 字节 | 用途 |
|---|---|---|
| fp32 | 4 | 优化器主状态、SSM 状态 |
| bf16 / fp16 | 2 | 默认训练/推理权重与激活 |
| fp8 (e4m3) | 1 | 量化权重、部分激活 |
| int4 / nvfp4 / nf4 | 0.5(+scale) | 量化推理权重 |

## 安全余量为什么必要

- PyTorch 缓存分配器、内存碎片、临时缓冲会让实际峰值高于理论和 ~10–20%。
- flash-attn / 融合 kernel 会**降**激活;朴素实现会**升**。
- 估算目标是"**不 OOM**",所以宁可略高:默认 `--margin 1.1`,长 seq 训练建议 1.2。
- 拿不准时:先用 batch=1、seq 短跑一步看 `torch.cuda.max_memory_allocated()`,再线性外推。
