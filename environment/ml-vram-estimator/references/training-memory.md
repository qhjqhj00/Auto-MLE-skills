# 训练显存:优化器 + 梯度 + 激活

> last updated: 2026-01。训练比推理吃显存得多。两大头:**优化器状态**和**激活**。

## 优化器状态(全量微调的最大头)

混合精度训练每参字节(常见约定):

| 优化器 | fp32 主权重 | 状态 | 合计每参 | 备注 |
|--------|------------|------|----------|------|
| **AdamW** | 4 | 动量 4 + 方差 4 | **12** | 默认;最贵 |
| SGD+momentum | 4 | 动量 4 | 8 | |
| Adafactor | 4 | 因子化 ≈ 0 | ~4 | 省显存,精度略让步 |
| 8-bit Adam (bnb) | 4 | 2+2 | ~8 | bitsandbytes,需支持的平台 |

加上权重 bf16(2)+ 梯度 bf16(2):**全量 AdamW ≈ 16 字节/参**。
→ 7B≈112GB,13B≈208GB,70B≈1.1TB(权重+梯度+优化器,还没算激活)。

## 激活内存(长 seq 时爆炸)

无梯度检查点时,每层都要存前向激活供反向用:

```
激活 ≈ batch × seq × hidden × 层数 × 系数(混合精度 ~16) + 注意力分数矩阵
注意力分数 ≈ batch × heads × seq² × 字节 × (full_attn层 / 总层)   ← seq² 项!
```

- **seq² 项**:序列翻倍,注意力激活涨 4 倍。长 context 训练的头号 OOM 元凶。
  flash-attn 不实体化这个 seq² 矩阵 → 大幅省激活(估算时若用 flash-attn 可下调)。
- 脚本对激活给保守近似;实测一步最准。

## 梯度检查点 (gradient checkpointing / activation recompute)

只存层边界激活,反向时重算其余:
- 激活内存从 O(层数) 降到约 **2× batch×seq×hidden×层数**(脚本里 grad-ckpt 的建模)。
- 代价:多一次前向,**约 +30% 训练时间**。
- 几乎总是值得开,尤其长 seq / 大模型。`--grad-ckpt`。

## LoRA / QLoRA(消掉优化器那一大块)

只训练少量低秩适配器参数,**冻结的基座权重无梯度、无优化器状态**:
- 优化器状态从"按全参算"降到"按 LoRA 参算"(常小几百倍)→ 这一项几乎归零。
- **QLoRA**:基座权重 4bit 量化常驻 + LoRA 适配器训练。7B QLoRA 可在 ~10–16GB 跑起来。
- 估算 LoRA:权重(可量化)+ 极小的梯度/优化器(只算适配器)+ 激活。是 OOM 时的强力退路。

## 其它省显存项

| 手段 | 砍哪一项 | 代价 |
|------|---------|------|
| 梯度检查点 | 激活 | +30% 时间 |
| LoRA/QLoRA | 优化器+梯度 | 表达力受限 |
| 8-bit / paged 优化器 (bnb) | 优化器 | 需支持平台 |
| FSDP/ZeRO-3 | 权重+梯度+优化器(切到多卡) | 通信开销,见 parallelism-memory.md |
| CPU/NVMe offload (ZeRO-Offload) | 优化器/权重挪出显存 | 慢,最后手段 |
| 降 batch + 梯度累积 | 激活 | 等效大 batch,步数变多 |

## 训练 OOM 排查顺序

1. 看 breakdown 哪项最大。优化器大 → LoRA/ZeRO;激活大 → grad-ckpt/降 seq;权重大 → 量化/分片。
2. 先开 grad-ckpt + 降 batch(最便宜),再考虑 LoRA,再考虑多卡分片。
3. 真要全量训大模型:FSDP/ZeRO-3 across N 卡 是标准答案。
